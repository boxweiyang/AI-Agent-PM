```sql
-- 性能指标要求

-- 1. 简单查询（带索引）
SELECT * FROM projects WHERE id = ?;
-- 执行时间：< 10ms
-- 扫描行数：1 行

-- 2. 条件查询（复合索引）
SELECT * FROM tasks
WHERE project_id = ? AND status = ? AND assignee_id = ?
ORDER BY priority DESC, created_at DESC
LIMIT 20;
-- 执行时间：< 50ms
-- 扫描行数：< 100 行

-- 3. 关联查询（使用连接）
SELECT p.name, u.name as owner_name, COUNT(t.id) as task_count
FROM projects p
LEFT JOIN users u ON p.owner_id = u.id
LEFT JOIN tasks t ON p.id = t.project_id
WHERE p.status = 'in_progress'
GROUP BY p.id, u.id
HAVING COUNT(t.id) > 0
ORDER BY task_count DESC
LIMIT 20;
-- 执行时间：< 200ms
-- 扫描行数：< 1000 行

-- 4. 统计查询（定时缓存）
SELECT
    DATE(created_at) as date,
    COUNT(*) as total_tasks,
    SUM(CASE WHEN status = 'done' THEN 1 ELSE 0 END) as completed_tasks
FROM tasks
WHERE project_id = ?
GROUP BY DATE(created_at)
ORDER BY date DESC
LIMIT 30;
-- 执行时间：< 500ms（实时）
-- 缓存后：< 50ms
-- 刷新频率：5 分钟
```

### 4.2 安全性需求

#### 4.2.1 认证安全

**密码安全：**

```
密码策略：
- 最小长度：8 位
- 复杂度要求：包含大小写字母、数字、特殊字符中的至少 3 种
- 禁止使用：用户名、邮箱、连续数字、常见弱密码
- 密码历史：不能与最近 5 次使用的密码相同
- 有效期：90 天强制更换
- 加密存储：bcrypt + salt（cost factor=12）

示例（符合要求的密码）：
✓ Aa123456!
✓ Secure@Pass2024
✓ MyP@ssw0rd#2024

示例（不符合要求的密码）：
✗ 123456（太简单）
✗ password（常见弱密码）
✗ zhangsan123（包含用户名）
✗ 12345678（纯数字）
```

**登录安全：**

```
登录流程安全措施：

1. 传输加密：
   - 全站 HTTPS（TLS 1.3）
   - 密码前端加密（RSA 公钥加密）
   - 防止中间人攻击

2. 防暴力破解：
   - 同一账号失败 5 次，锁定 30 分钟
   - 同一 IP 失败 20 次/小时，封禁 24 小时
   - 图形验证码（失败 3 次后弹出）

3. 会话管理：
   - JWT Token 有效期：2 小时
   - Refresh Token 有效期：7 天
   - Token 黑名单机制
   - 支持单设备登录（可选）

4. 登录日志：
   - 记录登录时间、IP 地址、设备信息
   - 异常登录提醒（异地登录、新设备）
   - 支持查看和管理登录设备
```

# 数据脱敏规则示例

class DataMasking:
    """数据脱敏工具类"""

    @staticmethod
    def mask_phone(phone: str) -> str:
        """手机号脱敏：138****1234"""
        return phone[:3] + '****' + phone[-4:]

    @staticmethod
    def mask_email(email: str) -> str:
        """邮箱脱敏：zha***@example.com"""
        if '@' not in email:
            return email
        username, domain = email.split('@')
        masked_username = username[:3] + '***' if len(username) > 3 else '***'
        return f"{masked_username}@{domain}"

    @staticmethod
    def mask_id_card(id_card: str) -> str:
        """身份证号脱敏：110101********1234"""
        return id_card[:6] + '********' + id_card[-4:]

    @staticmethod
    def mask_bank_card(card_no: str) -> str:
        """银行卡号脱敏：6222 **** **** 1234"""
        return card_no[:4] + ' **** **** ' + card_no[-4:]

// 前端 XSS 防护

// 1. 输入过滤
function sanitizeInput(input) {
  // 移除危险标签
  const dangerousTags = ["script", "iframe", "object", "embed", "form"];
  let sanitized = input;

  dangerousTags.forEach((tag) => {
    const regex = new RegExp(`<${tag}[^>]*>|<\/${tag}>`, "gi");
    sanitized = sanitized.replace(regex, "");
  });

  // HTML 实体编码
  const map = {
    "&": "&amp;",
    "<": "&lt;",
    ">": "&gt;",
    '"': "&quot;",
    "'": "&#039;",
  };

  return sanitized.replace(/[&<>"']/g, (m) => map[m]);
}

// 2. 输出编码
// Vue 自动进行 HTML 实体编码，防止 XSS
// 使用 v-html 时需要手动过滤

**CSRF 防护：**

```javascript
// CSRF Token 机制

// 1. 服务端生成 Token
@app.get("/api/csrf-token")
async def get_csrf_token():
    token = generate_csrf_token()
    # 存储到 session 或 Redis
    redis.set(f"csrf:{session_id}", token, ex=3600)
    return {"csrf_token": token}

// 2. 前端携带 Token
axios.interceptors.request.use(config => {
  const csrfToken = localStorage.getItem('csrf_token');
  if (csrfToken && config.method !== 'get') {
    config.headers['X-CSRF-Token'] = csrfToken;
  }
  return config;
});

// 3. 服务端验证
@app.middleware("http")
async def verify_csrf(request: Request, call_next):
    if request.method in ["POST", "PUT", "DELETE"]:
        token = request.headers.get("X-CSRF-Token")
        if not verify_csrf_token(token):
            raise HTTPException(status_code=403, detail="CSRF token invalid")
    return await call_next(request)
```

#### 4.4.2 系统稳定性

**MTBF（平均无故障时间）：**

```
目标值：> 720 小时（30 天）

计算方式：
MTBF = 总运行时间 / 故障次数

例如：
- 月度总运行时间：30 × 24 = 720 小时
- 月度故障次数：< 1 次
- MTBF > 720 小时
```

**MTTR（平均修复时间）：**

```
目标值：< 1 小时

计算方式：
MTTR = 总修复时间 / 故障次数

例如：
- 月度总修复时间：< 1 小时
- 月度故障次数：1 次
- MTTR < 1 小时
```

#### 5.2.2 反馈机制

**成功反馈：**

```
操作成功的反馈序列：
1. 按钮显示 loading 状态（0.3s）
2. 显示成功提示（绿色对勾动画，0.5s）

5. 保留已填内容
```
