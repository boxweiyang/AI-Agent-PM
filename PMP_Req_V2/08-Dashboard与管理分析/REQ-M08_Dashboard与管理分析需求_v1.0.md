# REQ-M08 Dashboard 与管理分析需求（v1.0）

---

## 1. 模块定位

为 **项目负责人 / 核心成员** 提供 **项目级一览**：把分散在 **迭代与 Story（M03）、Task 看板（M04）、人力与预订（M05）、变更（M06）、提测与缺陷（M07）** 中的关键数字与风险，收敛到 **少量卡片 + 可下钻列表**，**不接 BI 平台、不做自定义报表引擎**。

**V1 原则**：**只读聚合 + 跳转源模块**；编辑与状态变更仍在各业务页完成。

---

## 2. 与上下游模块的关系

| 上游（数据来源） | 本模块 | 下游 |
|------------------|--------|------|
| M03 迭代 / Story | 进度类指标、范围卡片 | 点击跳转迭代或 Story 列表（带筛选） |
| M04 Task | 按状态计数、阻塞、未分配 | 跳转看板或分配工作台（带筛选） |
| M05 人力池 / 预订 | 池人数、本迭代预订与已领 Task 对比（若有） | 跳转人力与预订页 |
| M06 CR | 未关闭 CR 数、按状态分布 | 跳转 CR 列表 |
| M07 提测 / 缺陷 | 进行中提测、打开缺陷、按严重程度 | 跳转提测/缺陷列表 |
| M02C（可选） | 接口 **已确认数 / 总数**（若 M02C 有状态字段） | 跳转接口管理 |

**不做（V1）**：跨项目组织级驾驶舱、复杂图表配置、定时邮件报表、SQL 级自定义查询。

**下游**：项目生命周期末端的 **结项、复盘、知识索引** 见 **REQ-M10**。

## 3. 入口与布局

### 3.1 入口

- **项目首页默认** 即 **本项目 Dashboard**（与「项目设置」等并列时，**概览** 为默认 Tab）。  
- **登录后多项目「工作台」** 的 **路由、壳布局、列表权限（管理员/可建项目/仅成员）** 见 **REQ-M09**；本文 §6 仅规定 **卡片上展示哪些摘要指标**。  
- 全局 **`工作台`**：项目卡片展示 **少量摘要指标**（见 §6），**点击进入** 本项目 Dashboard。

### 3.2 布局（V1）

- **顶栏**：**迭代筛选器**（默认 **当前迭代**；可选 **全部迭代**、指定迭代）  
- **主体**：**卡片网格**（2～4 列响应式）；每张卡片 **数字 + 短标签 +「查看」**  
- **可选区块**：**风险/关注列表**（合并：阻塞 Task、高优未关闭缺陷、逾期未关闭 CR——规则见 §5）

## 4. 指标卡片（V1 冻结清单）

以下均为 **在选定迭代范围下** 的统计（选「全部迭代」时按项目全量，文案需标明）。

| 卡片 | 指标说明 | 下钻目标 |
|------|----------|----------|
| **迭代进度** | Story 数、已完成 Story 数（完成定义：Story 下 Task 全部 `已完成`，或 Story 状态为完成——**与 M03 实现字段对齐，二选一写死**） | M03 Story 列表 |
| **Task 执行** | 各状态数量（待办/进行中/待测试/已完成）；**未分配负责人** 数 | M04 看板或列表（带筛选参数） |
| **阻塞** | 处于阻塞标记的 Task 数 | M04 阻塞列表 |
| **人力负荷** | 人力池人数；本迭代 **有预订** 的成员中，**已领 Task 数 / 预订** 的简要对比（仅展示，规则同 M05） | M05 |
| **变更 CR** | 未关闭 CR 数；可选 **待评审** 高亮 | M06 列表 |
| **质量** | **打开缺陷** 数；**致命+高** 单独数字；**进行中提测** 数 | M07 |
| **接口（可选）** | 已确认接口 / 接口总数 | M02C |

> **实现提示**：「当前迭代」缺省规则——取 M03 中标记为 **进行中** 的迭代，若无则取 **排序最前且未结束** 的迭代；具体由实现与 M03 字段统一，本文要求 **产品界面可看懂当前统计范围**。

## 5. 风险与关注列表（V1）

单一列表，按类型混排，**最多展示 N 条**（如 20），**查看更多** 跳转对应模块。

| 类型 | 纳入条件（示例） |
|------|------------------|
| Task | 有 **阻塞** 且未 `已完成` |
| 缺陷 | 严重程度 **致命/高** 且未 **已关闭** |
| CR | 状态为 **待评审** 或 **已采纳** 且长期未 **已关闭**（「长期」V1 可配置为 ≥14 天或固定提示） |
| 提测 | 状态 **已驳回** 且关联 Task 仍开放 |

排序：**致命缺陷 > 阻塞 Task > 待评审 CR > 其它**（实现可微调，需稳定可解释）。

---

## 6. 多项目「工作台」摘要（V1）

每张项目卡片至少展示：

- 项目名称、当前迭代名称（若有）  
- **打开缺陷数**、**阻塞 Task 数**、**待评审 CR 数**（三选二亦可，但需与 Dashboard 语义一致）

---

## 7. AI 辅助（按需，与全站一致）

- 抽屉：**`生成本周项目进展摘要`**：基于当前筛选范围与卡片数据生成 **短文**（邮件/周会用），**不写入业务数据**；用户 **复制** 即可。

---

## 8. 权限（V1）

- **项目成员** 可见本项目 Dashboard。  
- **非成员** 不可见（与项目 ACL 一致）。  
- **组织级** 汇总 **不做**（V2）。

---

## 9. 验收标准（产品口径）

- 进入项目默认可见 **Dashboard**，且 **迭代筛选** 改变后数字随之变化  
- 每张 **核心卡片**（Task、缺陷、CR、提测）可 **下钻** 到对应模块且 **带合理筛选**  
- **风险列表** 与手工在各模块核对 **一致**（同一数据集）  
- 不引入第二套状态定义；所有口径 **引用各模块已有状态**
