# REQ-M04 Task 与执行协作需求（v1.0）

---

## 1. 模块定位

承接 **REQ-M03** 产出的 **Task**（已具备标题、类型建议、优先级 0–4、排序等），在本模块完成：

1. **从人力池（REQ-M05）选人**，将 Task **指派**给具体成员（可后续调整）  
2. **执行期协作**：看板、状态流转、阻塞记录、开发与测试日常工作入口  
3. **一键生成 AI 提示词**（Dev / Test），便于在外部或内置 AI 中继续工作（与此前全站交互规范一致）

---

## 2. 与上下游模块的关系

| 上游 | 本模块 | 下游 |
|------|--------|------|
| REQ-M03 Task 草案 | 分配负责人、进入执行 | REQ-M06 变更请求 CR；REQ-M07 提测/缺陷/联调；**REQ-M08** 进度类数据源 |
| REQ-M06 CR | 采纳后可生成或关联 Task | Task 详情展示关联 CR |
| REQ-M05 人力池 | **选人**、负荷/预订提示 | — |
| REQ-M02C 接口 | Task 可关联接口，提示词中带契约摘要 | REQ-M07 联调视图、缺陷引用 |

---

## 3. 推荐流程（产品顺序，冻结）

```
M03 保存 Task 列表（可无负责人）
    → M05 维护人力池 + 可选迭代预订
    → M04 在「分配视图/看板」从池中选人绑定 Task
    → 开发与测试在看板/详情中推进状态 + 使用「一键提示词」
```

> 说明：也允许在 **M03 拆分 Task 时** 顺手选负责人（若已实现）；**规范上**仍以 **M05 池 + M04 分配** 为主流程，避免未入池人员被指派。

## 4. Task 分配

### 4.1 分配入口

- **迭代 → Story → Task 列表** 上每行 **`分配`** 或 **批量分配**
- **看板** 上卡片 **指派负责人**（同一人力池选择器）

### 4.2 人力池选择器

- 数据源：**REQ-M05** 项目人力池（默认排除「不参与排期」）
- 展示：姓名、角色、技能标签、**本迭代已领 Task 数**、**预订容量**（若有）
- 支持 **清空负责人**、**改派**

### 4.3 AI 辅助分派（可选）

- 抽屉：`AI 根据 Task 描述、技能、接口、预订推荐负责人`
- 主区为 **推荐结果列表**（Task→建议人选），用户 **勾选应用** 或 **逐条忽略**

---

## 5. 看板与状态

### 5.1 看板列（V1 建议）

- **待办 / 进行中 / 待测试 / 已完成**（可按项目模板简化为 **待办 / 进行中 / 已完成**）

### 5.2 操作

- **拖拽** 改变状态与 **同列排序**（与 M03 优先级/排序规则兼容）
- **阻塞**：填写阻塞原因、可选预计解除时间；在看板与列表 **醒目标识**

### 5.3 字段延续 M03

- 优先级 **0–4**（越小越高）、与 M03 一致  
- 关联 Story、迭代、可选关联 **接口（M02C）**

### 5.4 状态流转规则（Task 内部，V1）

- **主路径**：`待办` → `进行中` → `待测试` → `已完成`（若项目模板无「待测试」，则为三步）
- **拖拽**：允许在相邻或同组列间拖拽；**回退**（如 `已完成` → `进行中`）**允许**，建议 **二次确认** 或仅 **项目负责人 / Task 负责人** 可操作（实现层二选一，产品默认 **不禁止回退**）
- **跳过列**：允许 `待办` 直接拖至 `待测试`（例如纯文档类 Task），**不强制**经过每一列
- **与 CR**：若 Task 由 **REQ-M06** 派生或已关联 CR，在详情页 **只读展示 CR 编号与链接**；**不因 CR 状态自动改 Task 状态**（避免隐式规则）

## 6. 一键生成 AI 提示词（Dev / Test）

### 6.1 入口

- Task **详情页** 按钮：**`一键生成 AI 提示词`**
- 子操作：**`开发向`** / **`测试向`**（或 Tab 切换）
- **`复制`**、**`打开 AI 抽屉`**（将提示词带入侧栏继续追问，可选）

### 6.2 提示词自动上下文（冻结）

- Task 标题、描述、验收标准（来自 Story AC 摘要）
- 所属项目、迭代、Story 链接
- 关联 **接口**（M02C）路径/方法/参数摘要
- 关联需求/设计文档锚点（若有）
- **关联 CR**（若有）：编号 + 标题 + 废止/生效摘要（来自 REQ-M06）
- 当前状态、阻塞说明（若有）

### 6.3 输出形态

- 结构化 Prompt 文本（用户可复制到任意大模型）
- 不强制在本系统内完成对话（与「管理系统为主」一致）

---

## 7. 页面信息架构（建议）

- **项目 / 迭代 / 看板**（默认入口）
- **分配工作台**（表格：未分配 Task 优先筛选）
- **Task 详情**（评论可选 V2、附件可选 V2）；可选入口 **`发起提测`**（见 REQ-M07）

---

## 8. 业务规则（V1）

- **未分配负责人** 的 Task 允许存在；看板 **弱提示**「未分配」
- 指派对象 **必须在人力池内**（若从成员表同步池，则先加入池再指派）
- 删除 Task：**二次确认**；若已有工时/评论（未来模块）再叠加规则

---

## 9. 验收标准（产品口径）

- 可从 **人力池** 为 Task **单选/批量** 分配与改派
- 看板 **拖拽** 状态与排序可用
- **一键生成 AI 提示词**（开发/测试）可用且上下文完整
- 与 M03、M05、M02C 的关联在界面上 **可理解、可跳转**
