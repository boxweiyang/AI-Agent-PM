# REQ-M12 通用规范与验收标准（v1.0）

---

## 1. 模块定位

规定跨模块 **命名、编号、版本、权限与数据隔离** 的 **通用约定**，以及 **版本级验收** 的 **抽样口径**；各业务模块的 **详细功能验收** 仍以 **各分册 §验收** 为准，本文 **汇总索引 + 全局底线**。

---

## 2. 文档与工件命名（建议）

| 类型 | 建议格式 | 示例 |
|------|----------|------|
| 需求总纲 | `REQ-MASTER_{项目}_vX.Y` | REQ-MASTER_商城_v1.0 |
| 模块需求 | `REQ-MOD-{编号}_{名称}_vX.Y` | REQ-MOD-01_订单_v1.1 |
| 变更请求 | `CR-{序号}`（项目内递增） | CR-012 |
| 缺陷 | `BUG-{序号}` | BUG-045 |
| 提测记录 | `ST-{序号}` | ST-003 |

版本号 **主版本.次版本**；**破坏性变更** 升主版本，由项目约定。

---

## 3. 需求追溯（可选 V1）

- 功能开发可 **在内部跟踪** 关联 **Story ID / Task ID**（来自 M03/M04）
- **不强制** 每条代码提交带 REQ 号；若做 CI 钩子为 V2

---

## 4. 权限与数据隔离（全局底线）

- **项目级隔离**：用户 **仅能** 访问其为成员的项目（**系统管理员** 例外，见 REQ-M09）
- **操作与菜单**：与 **角色/权限位** 一致；**前端隐藏 + 后端鉴权** 双保险
- **审计**：关键操作（结项、删除模块文档、回滚版本）记录 **操作者、时间**（粒度由实现定，V1 至少 **结项与硬删**）

---

## 5. 非功能基线（V1 建议）

| 维度 | 基线 |
|------|------|
| 浏览器 | 最近两个大版本的 **Chrome / Edge / Safari**（或项目启动时书面锁定） |
| 可用性 | 核心路径 **无 JS 报错白屏**；接口失败有 **明确提示** |
| 性能 | 列表首屏 **可交互** 时间目标由项目排期单列（本文不写死毫秒数） |
| 无障碍 | V1 **不强制** WCAG 2.1 AA；**关键按钮** 具备可读标签即可 |

---

## 6. 版本发布验收（冒烟清单，抽样）

以下为 **发版前建议过一遍** 的 **最小集合**（细节以各模块验收为准）：

- [ ] **M09**：登录后首页 **权限正确**；进项目后侧栏 **正常**
- [ ] **M01**：可创建项目；无权限用户 **无法新建**
- [ ] **M02**：文档可保存、**版本历史与 Diff** 可用；**关联 CR** 可选保存
- [ ] **M03**：迭代/Story/Task 列表可保存；AI **应用/撤回**（若开启）可用
- [ ] **M04**：看板拖拽、分配、**一键提示词** 可复制
- [ ] **M05**：人力池维护、预订（若用）
- [ ] **M06**：CR 状态流转、关联文档版本
- [ ] **M07**：提测、缺陷 **闭环**
- [ ] **M08**：Dashboard 数字与下钻 **一致**
- [ ] **M10**：结项与复盘 **可保存**（若本版本包含）
- [ ] **M11**：各 AI 入口 **应用/撤回** 行为符合规范；需 AI 的场景具备 **§2.5 三种模式** 中分册声明的适用子集（含 **外置回填** 降级时引导清晰）

## 8. 术语（简表，与总文档互补）

| 术语 | 含义 |
|------|------|
| 应用 | 用户确认将 AI 输出写入业务数据或正文 |
| 一键撤回 | 撤销最近一次（或选定）AI 应用批次并清理相关上下文 |
| 已结项 | 项目终态之一，写权限受限（见 M10/M01） |

总词汇表见 **REQ-MASTER** §3.4；本文 §8 为互补简表。
