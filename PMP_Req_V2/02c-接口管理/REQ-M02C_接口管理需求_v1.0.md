# REQ-M02C 接口管理需求（v1.0）

---

## 1. 模块定位

本模块提供 **接口结构化数据** 的维护能力，与 **接口设计文档**（在 REQ-M02B 技术设计工作台中编辑）配合使用：

- 设计文档侧重 **叙述、评审、版本与 Diff**
- 本模块侧重 **可检索、可编辑、可关联 Task/前后端** 的 **接口清单与字段级定义**

目标：Task 拆分与联调时，前后端使用 **一致的接口名称、路径、方法与参数**。

---

## 2. 与上下游模块的关系

| 上游 | 本模块 | 下游 |
|------|--------|------|
| REQ-M02 需求（已确认） | AI 生成接口设计文档初稿的上下文来源 | REQ-M03 迭代与 Story |
| REQ-M02B 接口设计文档 | **同步入口**：`生成/同步到接口管理` | REQ-M04 Task、REQ-M06 CR、REQ-M07 开发测试协同 |

- 接口设计文档的版本历史、Diff、回滚、AI 一键撤回等规则见 **REQ-M02B**；本模块描述 **同步之后** 的结构化数据与页面行为。

---

## 3. 接口设计文档（归属说明）

- 「接口设计」作为 **技术文档的一种**，仍在 **REQ-M02B 技术设计工作台** 中维护（独立文档或挂载于模块技术文档树）。
- 本模块 **不负责** 文档正文的富文本/Markdown 编辑；仅负责 **结构化接口库** 及与文档的 **同步关系**。

---

## 4. 从文档到结构化数据：同步规则（V1 冻结）

### 4.1 触发方式

- 用户在接口设计文档或技术工作台工具栏点击 **`生成/同步到接口管理`**
- 同步范围：当前 **已发布/已确认版本**（与 REQ-M02 版本策略对齐），或用户 **勾选的部分接口块**

### 4.2 默认策略

- **合并新增**：在结构化库中 **追加** 新接口；已存在记录则进入 **冲突处理**
- **冲突判定**：同一项目下 **HTTP 方法 + 路径**（或项目内约定的唯一键）相同即视为冲突
- **冲突处理**：**弹窗**让用户对每条冲突选择 **「跳过」** 或 **「覆盖本条」**，禁止静默覆盖

### 4.3 同步溯源

- 每次同步记录：**来源文档 ID、文档版本号、操作者、时间、冲突处理摘要**
- 结构化接口条目可展示 **「来自文档 vX.Y」** 链接，跳转回对应版本（只读或提示升级文档）

## 5. 接口管理页面（功能需求）

### 5.1 信息架构

- **列表页**：筛选（模块、标签、方法、路径关键字）、排序、批量操作（可选 V2）
- **详情/编辑页**（或侧滑抽屉）：单接口完整字段编辑

### 5.2 建议字段（V1）

| 字段 | 说明 |
|------|------|
| 接口显示名 | 中文或业务名 |
| HTTP 方法 | GET/POST/PUT/PATCH/DELETE 等 |
| 路径 | 如 `/api/v1/projects` |
| 所属模块 | 与需求/技术模块 ID 关联 |
| 摘要说明 | 一句话 |
| 请求参数 | 结构化列表：名、类型、必填、说明 |
| 响应体 | 结构化或 JSON Schema 文本（V1 可先表格+备注） |
| 错误码说明 | 可选 |
| 状态 | 草案/已确认/已废弃（V1 可简化为 草案/已确认） |

### 5.3 CRUD

- **新增**：手工新增接口条目（不强制先有文档）
- **编辑**：任意字段可改；保存时记 **结构化数据版本** 或操作日志（V1 可简化为操作日志）
- **删除**：**二次确认** 后硬删（与模块文档删除策略一致）；若被 Task 引用则 **警告** 仍允许删除或 **禁止删除**（二选一，V1 建议：**警告后仍允许**，由用户承担）

### 5.4 与 Task 的关联（需求预留）

- Task 可 **关联一个或多个接口 ID**（V1 可先支持「关联接口」下拉，深度在 REQ-M04 展开）

---

## 6. AI 辅助（与全站一致）

- **文档侧 AI**：在 REQ-M02B 的接口设计文档中完成（生成初稿、应用、撤回）
- **本模块**：可提供 **「根据当前结构化列表生成联调说明 Markdown」** 等次要能力（可选 V2）
- V1 核心：**文档 AI → 同步 → 本页 CRUD**

---

## 7. 业务规则（V1）

- 未同步到本模块：**不阻塞** 项目其它操作；在 **拆分涉及前后端联调的任务** 时 **非阻塞提示**（文案见 REQ-M02B 7.1）
- 接口路径+方法在同一项目内 **建议唯一**；保存时若重复 **阻止提交** 或 **警告**（V1 建议：**阻止提交**）

---

## 8. 验收标准（产品口径）

- 可从接口设计文档 **一键同步** 到本模块，冲突时 **弹窗处理**，无静默覆盖
- 列表与编辑 **增删改查** 可用
- 接口条目可 **关联模块**，并可追溯 **来源文档版本**
- 与 REQ-M02B 中接口设计文档的职责边界 **无混淆**（文档编辑在技术工作台，结构化数据在本模块）
