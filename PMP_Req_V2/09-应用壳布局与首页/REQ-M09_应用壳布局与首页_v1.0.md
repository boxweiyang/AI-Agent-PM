# REQ-M09 应用壳、布局与首页（v1.0）

---

## 1. 模块定位

规定 **整站单页应用（SPA）** 的 **技术选型、整体布局、侧边导航、登录后首页（工作台）** 及 **按权限展示的差异**；与 **REQ-M01** 项目角色、**REQ-M08** 项目内 Dashboard 分工明确：**M09 = 站级壳与项目入口**，**M08 = 进入某项目后的概览与分析**。

---

## 2. 技术约束（冻结）

| 项 | 约定 |
|----|------|
| 框架 | **Vue 3**（Composition API 由实现定） |
| UI 组件库 | **Element Plus** |
| 形态 | **单页面 Web（SPA）**，路由前端管理 |
| 布局组件 | 使用 Element Plus **Container / Menu** 等搭建 **侧边栏 + 主内容区**（顶栏可选，见 §3） |

> 本文不写工程化细节（构建、目录结构），仅约束 **产品可见结构** 与 **权限行为**。

---

## 3. 全局布局

### 3.1 结构

- **左侧：侧边栏导航**（固定或可折叠；窄屏可收折为抽屉，V1 可简化为始终固定宽度）
- **右侧：主内容区**（`router-view` 承载业务页）
- **顶栏（推荐）**：左侧可放 **Logo / 系统名**，右侧 **当前用户、退出、（可选）全局搜索**；**不替代**侧边栏主导航

### 3.2 导航层级

- **一级菜单（站级）**：**工作台（首页）**、**项目管理**（或「所有项目」仅管理员可见）、**系统设置**（仅管理员，可选 V1）、**个人设置**（可选）
- **进入具体项目后**：侧边栏切换为 **项目内菜单**（需求、迭代、看板、测试协同、Dashboard…，与各 REQ 模块对齐，详细菜单表可在实现阶段按模块拆；本文要求 **与权限联动隐藏无权入口**）

### 3.3 面包屑（推荐）

- 项目内页面：`工作台 > 项目名称 > 当前模块`（实现可简化）

---

## 4. 登录后首页（工作台）

### 4.1 统一容器

- 路由如 `/` 或 `/workbench`，即 **首页**。
- 首页 **主体为「项目卡片列表」**（Element Plus `el-card` 或表格，由实现选择）；每张卡片 **点击进入项目**，进入后默认落地 **REQ-M08 项目 Dashboard**（或项目概览页，与 M08 一致）。

### 4.2 权限与可见内容（冻结）

| 角色 / 权限 | 项目列表范围 | **新建项目** 入口 | 其它 |
|-------------|----------------|-------------------|------|
| **系统管理员** | **系统中全部项目**；可附 **全站级统计摘要**（项目总数、活跃数等，V1 可仅项目数） | **显示**，进入 **REQ-M01 立项/新建流程** | 可进入任意项目（审计/协助）；菜单中可保留 **「全部项目」** 与管理员专用入口 |
| **具备「创建项目」权限的用户** | **我创建的** ∪ **我作为成员参与的**（去重展示）；可对「我创建的」打标签区分 | **显示**，进入 M01 新建 | 与 M01 **项目负责人** 或独立权限位对齐：*默认「项目负责人」角色拥有创建权限；是否给其它角色开闸由系统配置（V1 可写死仅负责人可建）* |
| **开发 / 测试 / BA 等普通成员**（无创建项目权限） | **仅我参与的项目** | **不显示**（按钮隐藏；直接访问新建路由时 **403 或重定向首页并提示**） | 卡片上仍可展示 M08 摘要字段（若已实现），规则同 M08 |

> **与 REQ-M01 对齐**：V1 角色含 **项目负责人、BA、开发、测试**。本文 **新增站级角色「系统管理员」**（平台运维/超管），**不在项目成员枚举内**，但可进入所有项目。

### 4.3 空状态

- 列表为空时：**引导文案** +（若有权限）**去新建项目** 按钮；无权限则 **「联系管理员或项目负责人将你加入项目」**。

### 4.4 新建项目

- 入口：**首页右上角「新建项目」** + 侧边栏（若有）**项目管理 → 新建**
- 跳转：**REQ-M01** 立项表单页；创建成功后 **进入该项目** 或 **回工作台并高亮新项目**

---

## 5. 路由与权限

- **前端路由 meta**：声明 `roles` / `permissions`（与后端返回的权限模型一致）
- **导航守卫**：无权限访问某路由时 **重定向首页或 403 页**，并 **Toast 提示**
- **侧边栏菜单**：根据权限 **过滤生成**，避免「看得见点进去报错」
- **接口鉴权**：以后端为准；前端仅做体验层隐藏

---

## 6. 与 REQ-M08 的分工

| 页面 | 归属 | 说明 |
|------|------|------|
| 登录后 **多项目列表 / 工作台** | **REQ-M09（本文）** | 按 §4 分权限 |
| **进入某一项目后的 Dashboard** | **REQ-M08** | 迭代筛选、卡片、风险列表、下钻 |

M08 所述 **「全局工作台」** 的 **布局与路由容器** 以 **本文 §3～§4** 为准。

---

## 7. 验收标准（产品口径）

- SPA 为 **Vue 3 + Element Plus**，具备 **侧边栏 + 主内容区**
- 三类用户（**管理员 / 可建项目 / 仅成员**）登录后 **首页列表与「新建项目」显隐** 符合 §4.2
- 无新建权限用户 **无法通过菜单或路由** 进入立项页（或被拦截提示）
- 从首页 **进入项目** 后，侧栏切换为 **项目内导航**，且与权限一致
