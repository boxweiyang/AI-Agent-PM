# REQ-M02D 数据库结构管理需求（v1.0）

---

## 1. 模块定位

本模块提供 **数据库表与字段** 的结构化维护能力，与 **数据库设计文档**（在 REQ-M02B 技术设计工作台中编辑）配合使用：

- 设计文档侧重 **说明、评审、版本与 Diff**
- 本模块侧重 **可编辑的库表模型** 与 **一键 DDL 导出**（SQLite / SQL Server / MySQL）

目标：与接口层、需求实体 **命名对齐**，降低 Task 拆分与前后端理解成本。

---

## 2. 与上下游模块的关系

| 上游 | 本模块 | 下游 |
|------|--------|------|
| REQ-M02 需求（已确认） | AI 生成库表初稿的上下文来源 | REQ-M03、REQ-M04 |
| REQ-M02B 数据库设计文档 | **同步入口**：`生成/同步到数据库结构管理` | 开发建库、联调、测试数据准备 |

- 数据库设计文档的版本历史、Diff、回滚、AI 一键撤回等规则见 **REQ-M02B**。

---

## 3. 数据库设计文档（归属说明）

- 「数据库设计」作为 **技术文档的一种**，在 **REQ-M02B** 中维护。
- 本模块负责 **结构化模型** 与 **DDL 生成**，不负责长文档编辑。

---

## 4. 从文档到结构化数据：同步规则（V1 冻结）

### 4.1 触发方式

- 用户点击 **`生成/同步到数据库结构管理`**
- 范围：当前 **已发布/已确认版本** 或用户勾选的表定义块

### 4.2 默认策略

- **合并新增** + **冲突弹窗**（同 **表名** 或 表内 **同字段名** 视为冲突，具体规则实现层可细化）
- 用户对每条冲突选择 **「跳过」** 或 **「覆盖本条」**，禁止静默覆盖

### 4.3 同步溯源

- 记录来源文档 ID、版本、操作者、时间
- 表/字段可展示 **「来自文档 vX.Y」** 链接

---

## 5. 数据库结构管理页面（功能需求）

### 5.1 信息架构

- **库/表树** 或 **表列表**（V1 可单库单 schema，多库 V2）
- **表详情**：字段列表 + 表级属性
- **字段编辑**：行内编辑或子表表单

### 5.2 建议字段（V1）

**表级**

| 字段 | 说明 |
|------|------|
| 逻辑表名 | 业务名 |
| 物理表名 | 如下划线命名 |
| 说明 | 可选 |
| 主键说明 | 可选 |

**字段级**

| 字段 | 说明 |
|------|------|
| 字段名 | 物理列名 |
| 类型 | 逻辑类型，映射到各方言 |
| 可空 | 是/否 |
| 默认值 | 文本或空 |
| 主键/唯一 | 标记 |
| 外键 | 引用表.列（可选） |
| 说明 | 可选 |

### 5.3 CRUD

- **新增表 / 新增字段 / 编辑 / 删除表或字段**
- 删除：**二次确认** 后硬删；若被接口或其它对象引用则 **警告**（V1 建议：**警告后仍允许**）

## 6. 一键生成 DDL

### 6.1 入口

- 页面级按钮：**`生成 DDL`**
- 可选子集：**当前选中表** / **全项目表**

### 6.2 方言选择（V1 冻结）

用户必选其一：

- **SQLite**
- **SQL Server**
- **MySQL**

### 6.3 生成内容

- **主路径**：`CREATE TABLE` 语句（含列定义、主键；外键/索引按 V1 能力可选生成）
- **可选勾选**（按方言显示可用项）：
  - 是否包含 **`CREATE DATABASE` / 文件库说明**（SQLite 可为文件路径说明；MySQL/SQL Server 可选建库语句）

### 6.4 预览与导出

- 生成前/后：**脚本预览** 面板
- **一键复制** 到剪贴板
- **相对上次生成** 或 **相对当前结构化数据** 的 **Diff 预览**（V1 可简化为「与上次导出脚本」文本 Diff）

### 6.5 类型映射

- 产品层要求：逻辑类型需 **可映射** 到三方言（具体映射表实现层维护；需求层只要求 **同一张表在三方言下可生成可执行脚本**）

## 7. AI 辅助
