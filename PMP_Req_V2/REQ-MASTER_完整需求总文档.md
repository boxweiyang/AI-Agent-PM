         → **提测/缺陷/联调(M07)** → **Dashboard/管理分析(M08)** → **收尾/复盘/知识(M10)** → …

## 3. 产品总述（与分册一致，已定稿）

### 3.1 产品愿景与目标用户

- **愿景**：用 **「管理系统 + 按需 AI」** 覆盖软件项目从 **立项 → 需求与设计文档 → 接口/库表 → 迭代与任务 → 人力与执行 → 测试与缺陷 → 度量与收尾** 的主链路，在可控、可追溯前提下 **减轻文档与排期成本**，让 **负责人、BA、开发、测试** 在同一套结构里协作。
- **半自动化定位**：AI **辅助生成与改写草案**，**不替代**人的确认；所有落库与版本变更以用户 **显式保存/应用** 为准（与 M11 一致）。
- **目标用户**：
  - **项目负责人**：立项、范围与节奏、看板与 Dashboard、结项与复盘。
  - **BA**：需求总纲与模块文档、与 Story/验收对齐。
  - **开发 / 测试**：任务执行、接口与库表消费、提测与缺陷、**一键 AI 提示词**（外置或大模型继续用）。
  - **系统管理员**（站级）：全项目可见、工作台与权限兜底（见 M09）。

### 3.2 范围：包含 / 不包含（V1）

| **包含（V1）** | **不包含（V1，可列 V2）** |
|----------------|---------------------------|
| 单团队 Web 产品内项目管理全流程（见 §1 分册表） | 替换 Jira/Azure DevOps 的全量能力、复杂工作流引擎 |
| 文档版本、Diff、回滚；需求/技术文档与 AI 协作 | 离线客户端、原生 App |
| 结构化接口库、库表与 DDL 导出（M02C/M02D） | 与外部缺陷/文档系统 **双向同步** |
| Task 看板、人力池、CR 台账、提测与缺陷、项目内 Dashboard | 组织级多项目 BI、自定义报表 SQL |
| Vue3 + Element Plus SPA 壳与分权限首页（M09） | CI/CD、流水线执行平台 |
| 全站 AI 交互统一规范（M11） | 模型选型与供应商锁定（实现阶段定） |

### 3.3 全局交互原则（摘录，权威见 M11）

- **管理系统为主**：列表 / 表单 / 文档为 **主界面**；AI **不**默认全屏打断。
- **按需抽屉**：用户点击 **`AI 辅助`** 等入口后展开；建议 → **`应用`** 后再写入；文档/列表类支持 **一键撤回本次 AI 修改** 并 **清理相关上下文**。
- **三种协作模式**（M11 §2.5）：**① 站内 AI 抽屉**；**② 跳过 AI、自行粘贴/上传**；**③ 外置任意 AI**：复制系统生成的提示词与输出格式约定 → 在外部模型生成 → **结构化粘贴回填** 并校验，作 **降级与偏好路径**。
- **统一文案与模式**：各模块 AI 入口、应用、撤回用语与 M11 **一致**，避免同一动作多种说法。

**详细**： [REQ-M11_全站AI交互规范_v1.0.md](11-全站AI交互规范/REQ-M11_全站AI交互规范_v1.0.md)

### 3.4 术语简表

| 术语 | 含义 |
|------|------|
| 总纲 / Master | 项目级总需求或总技术文档（L1） |
| 模块文档 | 按模块拆分的需求或设计文档（L2） |
| Story | 可交付用户价值的工作单元，下挂 Task（M03） |
| Task | 执行单元；状态在看板流转（M04） |
| CR | 变更请求；台账 + 关联文档版本与 Task（M06） |
| Book | 迭代内对成员容量的预订（M05） |
| 应用 | 用户确认将 AI 输出写入正文或业务数据 |
| 一键撤回 | 撤销指定 AI 应用批次并清除绑定上下文 |
| 已结项 | 项目终态，写权限受限（M10/M01） |

### 3.5 状态机总览（简表）

> 各状态 **迁移细则与权限** 以对应分册为准；此处便于研发对齐域模型。

| 对象 | 主要状态（示例） | 分册 |
|------|------------------|------|
| 项目 | 立项流程中 / 进行中 / 暂停 / **已结项**（与 M01 枚举统一） | M01、M10 |
| Task | 待办 → 进行中 → 待测试（可选）→ 已完成；允许回退（规则见 M04） | M04 |
| CR | 草稿 → 待评审 → 已采纳/已拒绝 → 已实现 → 已关闭 | M06 |
| 缺陷 | 新建/打开 → 已修复/待验证 → 已关闭；分支已拒绝、重新打开（M07） | M07 |
| 提测记录 | 待测试 → 测试中 → 已通过/已驳回 | M07 |

### 3.6 全局非功能与验收口径

- **非功能底线**：浏览器支持、权限与 **项目数据隔离**、关键操作审计、核心路径可用性与错误提示等，以 **[REQ-M12](12-通用规范与验收标准/REQ-M12_通用规范与验收标准_v1.0.md)** 为准。
- **验收**：各模块 **「验收标准（产品口径）」** 为功能权威；发版前 **冒烟清单** 见 M12 §6，与分册 §验收 **配合使用**。
