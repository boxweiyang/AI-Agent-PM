# PMP_Req_V2

本目录用于存放全新版本的需求文档，与旧版 `PMP_Doc` 完全隔离，避免混淆。

## 文档规划

- `REQ-MASTER_完整需求总文档.md`：**总入口与分册导航**（最后阶段再补全总述；当前为索引 v0.1）
- `01-项目与立项/`
- `02-需求与文档协作/`
- `02b-架构与技术设计/`（技术/架构/设计文档工作台；可与开发并行）
- `02c-接口管理/`（结构化接口库、与接口设计文档同步）
- `02d-数据库结构管理/`（表/字段结构化、与库设计文档同步、DDL 导出）
- `03-迭代与Story管理/`
- `04-Task与执行协作/`
- `05-人力池与资源预订/`
- `06-变更与CR管理/`（开发期范围变更台账；与 M02 版本、M04 Task 联动）
- `07-开发与测试协同/`（原规划 M06 顺延）
- `08-Dashboard与管理分析/`
- `09-应用壳布局与首页/`（Vue3 + Element Plus SPA；侧边栏；分权限工作台）
- `10-收尾复盘与知识沉淀/`
- `11-全站AI交互规范/`
- `12-通用规范与验收标准/`

## 工作方式

1. 逐模块讨论并确认需求（功能、交互、边界、价值）。
2. 每个模块先出草案，再迭代到“已确认”。
3. 所有模块确认后，汇总生成最终总文档。
